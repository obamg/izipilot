1. Presentation d'IziPilot

1.1 Qu'est-ce qu'IziPilot ?

IziPilot est la plateforme de pilotage OKR (Objectives & Key Results) d'IziChange S.A. Elle remplace les fichiers Excel de revue hebdomadaire par une application web moderne, accessible depuis ordinateur et telephone.

Objectif principal : chaque lundi matin avant 9h00, chaque Product Owner (PO) saisit la progression de ses indicateurs cles. Le Management et le CEO disposent alors d'une vue consolidee en temps reel de l'avancement de l'entreprise.

Acces : https://izipilote.com — compatible Chrome, Firefox, Safari, Edge et navigateurs mobiles.

1.2 Les roles utilisateurs

Role Permissions Acces
C CEO Acces total : lecture, ecriture, administration, tous les OKRs Toutes les pages + Administration
M MANAGEMENT Synthese, alertes, decisions, lecture de tous les OKRs Dashboard, Synthese, Alertes, Historique, Actions
P PO Saisie hebdomadaire de son perimetre + lecture globale Dashboard, Ma revue, Historique, Actions
V VIEWER Lecture seule de toutes les donnees Dashboard, Historique (lecture seule)

1.3 Le vocabulaire OKR

Terme Definition
Objectif (O) Un but qualitatif ambitieux a atteindre sur un trimestre. Ex: "Augmenter la penetration mobile en Afrique de l'Ouest"
Key Result (KR) Un indicateur mesurable qui prouve l'atteinte de l'objectif. Ex: "Atteindre 10 000 utilisateurs actifs"
Score Pourcentage d'avancement du KR : valeur actuelle / cible. Arrondi a l'entier.
Revue hebdo Saisie du PO chaque lundi matin : progression, statut, bloqueurs, actions
Action Tache concrete rattachee a un KR, avec responsable, priorite et echeance

Code couleur des statuts OKR

Statut Condition Signification
En bonne voie Score ≥ 70% Le KR progresse normalement vers la cible
Attention Score entre 40% et 69% Progression insuffisante, vigilance requise
Bloque Score < 40% ou declare bloque Obstacle majeur, escalade obligatoire sous 48h
Non demarre Pas encore lance Le travail n'a pas encore commence

2. Prise en main

2.1 Connexion

Rendez-vous sur https://izipilote.com. La page de connexion s'affiche.

Entrez votre adresse email professionnelle (@izichange.com)
Entrez votre mot de passe
Cliquez sur "Se connecter"
Premiere connexion ? Votre compte a ete cree par l'administrateur. Contactez le CEO ou l'equipe IT si vous n'avez pas vos identifiants.
Securite : Votre session expire automatiquement apres 4 heures d'inactivite. Vous serez redirige vers la page de connexion.

2.2 L'interface principale

L'interface se compose de trois zones :

  • Barre laterale gauche : navigation, produits, departements, administration
  • En-tete : nom de la page, informations contextuelles
  • Zone principale : contenu de la page selectionnee

2.3 La barre laterale

La barre laterale est votre point d'entree vers toutes les fonctionnalites :

Navigation principale

MenuDescriptionRoles
DashboardVue d'ensemble des KPI et scoresTous
Ma revue hebdoFormulaire de saisie du lundiPO, Management, CEO
SyntheseVue consolidee tous produits/deptsManagement, CEO
HistoriqueCourbes de progression sur 13 semainesTous
AlertesAlertes actives et decisionsTous (resolution: Management, CEO)
ActionsSuivi des actions rattachees aux KRsTous

Produits et departements

Les sections Produits et Departements sont depliables. Chaque entite affiche son code (ex: P1, D3), son nom et son score global avec un indicateur de couleur.

Administration (CEO uniquement)

Le menu Administration n'est visible que pour le CEO. Il donne acces a la gestion des utilisateurs, produits, departements, OKRs et parametres de l'organisation.

3. Tableau de bord

Le tableau de bord est la page d'accueil apres connexion. Il offre une vue synthetique de l'etat de l'entreprise.

3.1 Les indicateurs cles (KPI)

Cinq cartes KPI s'affichent en haut de la page :

KPIDescription
Score global Moyenne ponderee de tous les scores OKR de l'organisation, affichee en gros chiffre avec code couleur
En bonne voie Nombre de KRs avec un score ≥ 70% ON_TRACK
Attention Nombre de KRs entre 40% et 69% AT_RISK
Bloques Nombre de KRs < 40% ou declares bloques BLOCKED
Actions Compteurs par statut : A faire, En cours, Bloquees, Terminees

3.2 Scores OKR par produit

La section principale affiche la liste des produits avec pour chacun :

  • Le code produit (badge colore, ex: P2)
  • Le nom du produit
  • Le score global en pourcentage
  • En depliant : la liste des Key Results avec barres de progression individuelles

3.3 Alertes actives

Le panneau de droite affiche les alertes non resolues. Chaque alerte montre sa severite (icone couleur), le KR concerne et un message descriptif. Cliquez sur une alerte pour acceder a la page Alertes.

Indicateur de soumission : L'en-tete du dashboard indique combien de POs ont soumis leur revue cette semaine (ex: "12/15 POs"). Le bouton "Soumettre ma revue" vous redirige vers la page de saisie.

4. Revue hebdomadaire (PO)

4.1 Le cycle du lundi

Deadline : chaque lundi a 9h00 (heure de Cotonou).
Chaque PO doit soumettre sa revue avant cette echeance. Passe ce delai, le systeme genere automatiquement une alerte "Saisie manquante".

Le lundi matin, si vous etes PO et que vous accedez au Dashboard avant 9h00, vous serez automatiquement redirige vers la page "Ma revue hebdo".

4.2 Saisir sa revue

La page de revue affiche tous les Key Results dont vous etes responsable, regroupes par objectif.

Pour chaque KR, remplissez :

ChampDescriptionObligatoire
Progression Curseur de 0 a 100% representant l'avancement reel Oui
Valeur actuelle Pour les KRs numeriques : la valeur concrete (ex: 4500 utilisateurs) Selon type KR
Statut En bonne voie Attention Bloque Non demarre Oui
Bloqueur Description du probleme bloquant (si statut = Bloque) Si bloque
Action requise Aide ou decision necessaire du Management Non
Commentaire Toute information complementaire utile Non
Marquage "Bloque" : Lorsque vous definissez un KR comme bloque, une alerte est automatiquement creee et envoyee au Management. Le champ "Bloqueur" devient obligatoire pour expliquer la situation.
Accedez a "Ma revue hebdo" via la barre laterale
Pour chaque KR, ajustez la progression et le statut
Remplissez le bloqueur si le KR est bloque
Ajoutez un commentaire si necessaire
Cliquez sur "Soumettre ma revue" en bas de page

4.3 Creer des actions

Sous chaque KR, une section "Actions" liste les actions existantes et permet d'en creer de nouvelles.

Pour creer une action :

  • Titre : description courte de l'action (obligatoire)
  • Description : details complementaires
  • Responsable : personne assignee (membres du departement en priorite)
  • Priorite : Basse, Moyenne, Haute ou Urgente
  • Echeance : date limite de realisation

4.4 Sauvegarde automatique

Le formulaire de revue sauvegarde automatiquement vos saisies dans le navigateur (localStorage). Si vous perdez la connexion ou fermez la page par erreur, vos donnees seront restaurees a la prochaine visite.

Note : La sauvegarde locale est un brouillon. Vos donnees ne sont envoyees au serveur qu'apres avoir clique sur "Soumettre". La sauvegarde locale est effacee apres soumission reussie.

6. Historique et courbes

La page Historique & courbes permet de visualiser la progression des scores sur une fenetre glissante de 13 semaines.

Utilisation

Selectionnez un type d'entite : Produit ou Departement
Choisissez l'entite specifique dans la liste
Optionnel : selectionnez un KR specifique pour voir son evolution detaillee

Le graphique affiche :

  • L'axe horizontal : les semaines (S01, S02, ... S13)
  • L'axe vertical : le score en pourcentage (0% a 100%)
  • Une courbe de tendance pour visualiser l'evolution
  • Les seuils de reference a 40% (rouge) et 70% (vert)
Regle Google OKR : Un score de 70% est considere comme une reussite. 100% signifie que les cibles etaient trop faciles. Un score < 40% pendant 3 semaines consecutives declenche une recommandation de recalibrage.

7. Alertes et decisions

7.1 Types d'alertes

Le systeme genere automatiquement des alertes selon ces regles :

TypeDeclencheurSeverite
KR Bloque Un KR est declare bloque par le PO ou son score passe sous 40% CRITIQUE si score < 40%
HAUTE si declare manuellement
KR en baisse Le score diminue pendant 2 semaines consecutives MOYENNE
Score < 40% Le score reste sous 40% pendant 3 semaines consecutives CRITIQUE
Saisie manquante Un PO n'a pas soumis sa revue pour la semaine en cours MOYENNE
Escalade 48h Un KR bloque n'a pas ete resolu apres 48 heures CRITIQUE

7.2 Resoudre une alerte

Accessible uniquement aux roles CEO et Management.

Accedez a la page Alertes & decisions
Filtrez par statut (Actives / Toutes / Resolues) ou par type
Cliquez sur "Decider" a droite de l'alerte
Redigez la resolution ou decision prise dans le champ texte
Cliquez sur "Confirmer"

Une fois resolue, l'alerte apparait avec un badge vert "Resolu" et la resolution est visible par tous.

7.3 Escalade 48h

Regle metier : Tout KR bloque non traite dans les 48 heures declenche une alerte d'escalade critique. Un cron quotidien a 10h00 (heure Cotonou) verifie les alertes KR_BLOCKED non resolues et cree l'escalade automatiquement.

9. Administration (CEO)

Cette section est accessible uniquement au CEO.

9.1 Gestion des utilisateurs

Page : Administration > Utilisateurs

Creer un utilisateur

Cliquez sur "Nouvel utilisateur"
Remplissez : email, nom complet, role (CEO/Management/PO/Viewer)
Validez. Un mot de passe temporaire est genere.

Actions disponibles

  • Modifier : changer le nom, email ou role
  • Reinitialiser le mot de passe : genere un nouveau mot de passe temporaire
  • Desactiver : bloque l'acces sans supprimer le compte

9.2 Gestion des produits

Page : Administration > Produits

Les produits s'affichent en grille de cartes. Chaque carte montre :

  • Code et nom du produit
  • Badge de statut (Actif, En developpement, Planifie, En pause)
  • Proprietaire (PO assigne)
  • Nombre d'objectifs rattaches

Creer / modifier un produit

  • Code : identifiant court (ex: P8)
  • Nom : nom complet du produit
  • Couleur : couleur d'identification dans l'interface
  • Statut : Actif, En developpement, Planifie, En pause
  • Proprietaire : PO responsable

9.3 Gestion des departements

Page : Administration > Departements

Meme structure que les produits, avec en plus la gestion des membres. Chaque membre peut avoir le role LEAD (responsable) ou MEMBER.

9.4 Gestion des OKRs

Page : Administration > OKRs

Creer un objectif

Cliquez sur "Creer un objectif"
Remplissez : titre, motivation ("Why"), entite (Produit ou Departement), trimestre
Validez

Ajouter un Key Result

Sous l'objectif, cliquez sur "Ajouter un KR"
Choisissez le type :
  • Numerique : valeur cible + unite (ex: 10 000 utilisateurs)
  • Pourcentage : cible en % (ex: 95% de satisfaction)
  • Date : date cible (avancement saisi manuellement par le PO)
  • Binaire : oui/non (ex: "Obtenir la licence")
Assignez un proprietaire (PO responsable)
Validez

Filtres

  • Par trimestre : Q1, Q2, Q3, Q4 ou tous
  • Par type d'entite : Produit, Departement, Entreprise ou tous

9.5 Parametres de l'organisation

Page : Administration > Organisation

Affiche les statistiques globales et permet de modifier :

  • Le nom de l'organisation
  • Le slug (identifiant URL)
  • La couleur principale de la marque

10. FAQ et support

Questions frequentes

Je ne peux pas me connecter

Verifiez que vous utilisez la bonne adresse email (@izichange.com). Si le probleme persiste, demandez au CEO de reinitialiser votre mot de passe via Administration > Utilisateurs.

Je ne vois pas la page "Ma revue hebdo"

Cette page n'est accessible qu'aux POs, Management et CEO. Les Viewers n'ont qu'un acces en lecture seule.

J'ai oublie de soumettre ma revue lundi

Vous pouvez toujours la saisir apres la deadline. Cependant, le systeme aura genere une alerte "Saisie manquante" visible par le Management. L'alerte sera automatiquement resolue apres votre soumission.

Comment signaler un bloqueur ?

Dans votre revue hebdomadaire, changez le statut du KR concerne en Bloque et decrivez le probleme dans le champ "Bloqueur". Une alerte sera automatiquement creee.

Je ne vois pas le menu Administration

Ce menu est reserve au CEO. Si vous avez besoin de modifications sur les utilisateurs, produits ou OKRs, contactez le CEO.

Puis-je utiliser IziPilot sur mobile ?

Oui, l'interface est optimisee pour les smartphones. La barre laterale se replie automatiquement et le formulaire de revue est en colonne unique pour faciliter la saisie tactile.
